Karenderia Mobile App

  • created: 11/23/2015
  • by: bastikikang
  • email: basti@codemywebapps.com

Karenderia Mobile App is ordering application for Karenderia Multiple Restaurant System

Features

Online Easy Ordering

  • Easy to search for restaurant
  • Filter restaurant by cuisine or by services
  • Auto detects location address
  • Multiple addons for food item
  • Multiple size and price for food item
  • Ingredients selections for food item
  • Cooking reference
  • Real time order status
  • Easy to re-order previous ordered food item

Payment Gateway

  • Cash on delivery
  • Paypal
  • Pay using credit cards on debit cards

Book a table

  • Easy booking table
  • Receive updates on your booking status

Restaurant Review

  • Add your own reviews to a restaurant

Android Push Notification

  • Customer can receive push notification about the status of their order

Installation

First you need to install the mobile app modules on your karenderia multiple restaurant system these will include the mobile api engine and mobile app settings.

  1. create a folder called modules on protected folder. the full path for example protected/modules
  2. copy the folder modules/mobileapp to protected/modules (mobileapp is the file where you download from Codecanyon)
  3. To see the mobile app settings,
    Login to admin panel edit your account (admin-> User list)
    under the User Access tick the Add-ons and MobileApp an save your account then logout and relogin again to see the addon menu

  4. After you have re-login you should see now the Mobile app settings section
    see screenshot below

  5. Now update KMRS with the tables for mobileapp, go to menu admin -> mobileapp -> Update Db Tables



    when you click update db tables it will open a new link showing new table and fields is created,
    take note you should see the word (FINISH) at the very end.
  6. Done

Configure Mobile App

  1. Set api url and API hash key
    • in mobile app addon module -> general settings copy the Your mobile API URL
    • in API hash key set any values without spaces like unique password and click save settings, after saving the settings copy your API hash key
    • now set the api url and hash key to your mobile app config, open the file www/js/config.js
      and set the value to ApiUrl and APIHasKey as show below

  2. Update config.xml
    • Open the file kmrsmobile/www/js/config.js
      change the following information to yours like id, name of app and description etc.
      Note: xml does not like special character like comma



    • To enabled google login to your app you need to set REVERSED_CLIENT_ID in your config.xml,
      you can skip this if you don't want google login available in your app.

      how to get reverse client id follow this steps https://codesundar.com/create-reverse-client-id-for-google-login/



    • Set facebook App ID, order for facebook login to work in your app
      you need to set your facebook APP ID ,
      take note : this is very important if you don't set any facebook app id your app will crash, if you don't know how to configure facebook app id just remove this plugin in your config.xml

      to setup facebook app id follow this steps https://developers.facebook.com/docs/android/getting-started



  3. Set Google api key for map to work in the app.
    if you already have google api key set in kmrs you can use that key for mobile app
    • go to your google developer account http://console.developers.google.com/
    • Create a project and on the dashboard click Enabled APIS and services

    • Enabled the following api

      Google Maps Distance Matrix API
      Google Maps Geocoding API
      Google Maps JavaScript API
      Google Places API Web Service
      Google Maps Directions API
    • Now go credentials and click credentials and select API key

    • copy the api key and open up the file www/index.html and add the key as show below

  4. Create the file google-services.json for Android Push Notification
    • go to firebase https://console.firebase.google.com
    • Click on add project and name the project as shown below

    • Once you are in the dashboard click on add firebase to your android app

      when entering package name take note the package name should match or the same in your config.xml id, in screenshot below the package name is com.krestaurant.app

    • download the google-services.json by clicking download config file as show below

      after download press next and skip

    • copy the file

      to mobile app folder as shown below

  5. Set Firebase Server Key to Mobile App Module
    • In your Firebase console click Project Overview -> Project settings,
      Under Cloud Messaging tab copy server key and paste it your mobile app addon -> General settings -> Firebase Clound Message ->Server Key

  6. Now we will add IOS Push notification, you can skip this part you don't want to compile the app to iOS
    • Click on Add another app and choose Add firebase to your iOS App

    • download the GoogleService-Info.plist and copied the file to Mobile App files

    • for iOS push to work you will need to upload push certificate , click Project Overview -> Project settings. you can generate push certificate in your apple developer account

  7. Now your ready to build to app to http://build.phonegap.com/
    • Zip the mobile app files as show below, note there is many ways to zip the file but the important part is you need to zip the file inside the folder,
      Note: do not zip folder kmrsmobile.

    • Log in to http://build.phonegap.com/ and upload the zip file as show below.

  8. Done

Cron jobs

to run cron jobs you need to set the cron jobs link in your cpanel, you can find the cron jobs
for processing push notification in admin -> mobileapp -> Push CronJobs

here is the sample video on how to run cron jobs https://youtu.be/WjndBGKXF7A

cron jobs usually send email report about the status
in order to remove this you need to add > /dev/null 2>&1 at the end of url
https://www.a2hosting.com/kb/developer-corner/linux/disabling-e-mail-notifications-from-cron-jobs

How to translate the app

1. to translate the backend copy the file modules/mobileapp/translation-file/mobile.php
to folder protected/messages/en
example if you want to translate to japanese create a folder like protected/messages/jp
and copy the file modules/mobileapp/translation-file/mobile.php to /jp folder
and from there translate all the words found in mobile.php

2. to translate mobile app files just copy the files modules/mobileapp/translation-file/mobileapp.php
to the messages folder protected/messages/en and from there translate all the words found in mobileapp.php

How to change notification ringtone

In latest version of mobileapp 2.1 up, you can simply change the notification ringtone
by replacing the file www/beep.wav
the format should be .wav file

Note: the filename should be beep.wav if you change it to any filename push ringtone will not work

Frequently Asked Questions

What version of KMRS this modules will run?
- Mobile App modules will run on version 2.4 and up,
if you are using old version contact me here

i compiled the app but it crashes
- you need to put facebook app id in your config.xml or if you don't need facebook just remove it from your config.xml
example 

 <platform name="android" >	
		<config-file parent="/resources" target="./res/values/strings.xml">
			<string name="fb_app_id">2049491728632651
			<string name="fb_app_name">android login
		</config-file>               
    </platform>
 
    <gap:plugin name="cordova-plugin-facebook4" source="npm"  spec="3.2.0">
	  <variable name="APP_ID" value="2049491728632651" />
	  <variable name="APP_NAME" value="android login" />
	</gap:plugin>

i run the app but keeps loading getting settings
- this issue is with your api make sure you updated the db and use the latest version of kmrs and modules to avoid error in your api

I upload the folder mobileapp to modules folder but blank screen in admin backend mobile app
- try to set the folder assets to 0777 permission (assets folder is the assets folder of kmrs)

How to change the mobile app logo on home section of the app?
- just replace the following files on www/css/images/banner.png (take note of the width and height of the image)

How to change the default logo for merchant and food item?
- go to mobile app modules on general settings -> Default Image upload your own logo

How to change the orange header background color on home section of the app?
- open the file www/css/app.css look for the css .orange-header under that change the background properties background:#f75d34;

How to change the app icon and splash screen
- replace the following files kmrsmobile/icon.png and kmrsmobile/splash.png
for the icon/splash screen make sure the it has a valid size according to different device
see this link for more information http://docs.build.phonegap.com/en_US/configuring_icons_and_splash.md.html

After compile hydration appears
You will need to disabled the phonegap Hydration
After logging in navigate to the apps page, and select your app by clicking its title or icon. On the app's detail page, open the "Settings" panel. Under the "Basic" header de-select the checkbox labeled "enable hydration", and finally hit save. You've now successfully disabled Hydration.

After compile and upload to build.phonegap the app does not connect to API
if you upload the zip file on build.phonegap the icon does not show or the app does not load, reason is maybe you zip the whole folder kmrsmobile this is not correct make sure you are inside the folder kmrsmobile and select all the files and add to zip

Installing the app return errors package is not valid
maybe you put some special charaters on config.xml , xml does not like special charaters like ? and ,

How to submit my app to apple store?
http://wiki.nsbasic.com/Submitting_to_the_iOS_App_Store

Timeout when getting location in android 6.0
Afetr you install the app and getting timeout when getting your current location restart your device and try again

Error parsing package
This is issue in android version 4.0 cause the paypal plugin will not work in lower version for android
you can run fixed it by modifying the config.xml
remove this line 

< gap:plugin name="com.paypal.cordova.mobilesdk" source="npm"  / >
and 
< preference name="android-minSdkVersion" value="16" / >
